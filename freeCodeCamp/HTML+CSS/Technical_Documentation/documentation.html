In object-oriented programming, a class is an extensible code pattern for creating objects that sets them to initial values ​​(properties) and implementations of behavior (methods).

In practice, we often need to create many objects of the same type, for example users, products or something else. As we already know from the Constructor chapter, the "new" operator can help with this new function. But modern JavaScript also has a more advanced "class" construct that provides new features useful for object-oriented programming.

Syntax class

The basic syntax looks like this:

                        
                            1       class MyClass {
                            2            // class methods
                            3           constructor()  { ... }
                            4           method1() { ... }
                            5           method2() { ... }
                            6           method3() { ... }
                            7           ...
                            8         }

Then use the call new MyClass()to create a new object with all the methods listed. In this case, the method is automatically called constructor(), in which we can initialize the object. For example:

When called new User("Ivan"):

  1. A new object is created.
  2. constructor runs with the given argument and stores it in this.name.
  3. You can then call methods on the object, such as user.sayHi().

Methods in a class are not separated by comma

A common mistake novice developers make is to put a comma between class methods, which leads to a syntax error.

Class syntax is different from object literals, do not confuse them. Commas are not required within classes.

What is a class?

So, what is it class? This is not a completely new linguistic entity, as it might seem at first glance. Let's take all the magic away and see what a class really is. This will help in understanding many complex aspects. In JavaScript, a class is a type of function.

Take a look:

                        
                            1       class User {
                            2           constructor(name) { this.name = name; }
                            3           sayHi() { alert(this.name); }
                            4         }
                            5
                            6         // proof: User - is a function
                            7         alert(typeof User); // function

Here's what the construct actually does class User {...}:

  1. Creates a function named User, which becomes the result of a class declaration. The function code is taken from the method constructor(it will be empty if there is no such method).
  2. Stores all methods such as sayHi, in User.prototype.

More than just syntactic sugar

It's sometimes said that class it's just "syntactic sugar" in JavaScript (syntax to improve code readability, but doesn't do anything fundamentally new), because we can do the same thing without the class: 

                        
                        1       // rewrite class User with pure functions
                        2
                        3       // 1. Create function constructor
                        4       function User(name) {
                        5         this.name = name;
                        6        }
                        7       // each function prototype has default property constructor,
                        8       // that's why we don't need to create it
                        9
                        10      // 2. add method to prototype
                        11      User.prototype.sayHi = function() {
                        12          alert(this.name);
                        13      };
                        14
                        15      // Usage:
                        16      let user = new User("Ivan");
                        17      user.sayHi();

The output of this code is very similar. So there are indeed reasons why class it might be considered syntactic sugar to define a constructor along with prototype methods.

Class Expression

Like functions, classes can be defined inside another expression, passed, returned, assigned, etc. Example Class Expression (similar to Function Expression): 

                        
                            1       let User = class {
                            2          sayHi() {
                            3            alert("Hi");
                            4          }
                            5        };

Similar to a Named Function Expression, a Class Expression can have a name. If a Class Expression has a name, then it is visible only inside the class: 

                        
                            1   // "Named Class Expression"
                            2      let User = class MyClass {
                            3       sayHi() {
                            4         alert(MyClass); // we can see name MyClass 5      only inside the class
                            6       }
                            7     };
                            8  
                            9     new User().sayHi(); // prints the definition MyClass
                            10  
                            11    alert(MyClass); // error, name MyClass is not visible outside the class

We can even dynamically create classes on demand:

                        
                            1       function makeClass(phrase) {
                            2           // declare class and return it
                            3       return class {
                            4         sayHi() {
                            5           alert(phrase);
                            6         };
                            7       };
                            8     }
                            9 
                            10    // Create a new class
                            11    let User = makeClass("Hi");
                            12  
                            13   new User().sayHi(); // Hi
Getters/setters, other abbreviations

Like literal objects, classes can declare computed properties, getters/setters, etc. Here is an example user.name implemented using get/set: 

                        
                            1       class User {
                            2
                            3       constructor(name) {
                            4         // calls the setter
                            5        this.name = name;
                            6        }
                            7  
                            8       get name() {
                            9         return this._name;
                            10      }
                            11  
                            12      set name(value) {
                            13        if (value.length < 4) {
                            14         alert("Name is too short.");
                            15          return;
                            16        }
                            17        this._name = value;
                            18      }
                            19 
                            20    }
                            21 
                            22    let user = new User("Ivan");
                            23    alert(user.name); // Ivan
                            24 
                            25    user = new User(""); // Name is too short.

When declaring a class, getters/setters are created on User.prototype, like this: 

                        
                            1       Object.defineProperties(User.prototype, {
                            2           name: {
                            3             get() {
                            4               return this._name
                            5              },
                            6             set(name) {
                            7               // ...
                            8             }
                            9           }
                            10        });

Example with a calculated property in parentheses [...]:

                        
                            1       class User {
                            2
                            3           ['say' + 'Hi']() {
                            4             alert("Hi");
                            5           }
                            6  
                            7         }
                            8  
                            9         new User().sayHi();
Class Properties

Older browsers may need a polyfill. Class properties were recently added to the language.

In the example above, the class User only had methods. Let's add a property: 

                        
                            1       class User {
                            2           name = "Anonymous";
                            3  
                            4           sayHi() {
                            5             alert(`Hi, ${this.name}!`);
                            6           }
                            7         }
                            8  
                            9         new User().sayHi();

The property name is not set to User.prototype. Instead, it is created by the operator new before running the constructor; it is precisely a property of the object.

Total

The basic syntax for classes looks like this:

                        
                            1       class MyClass {
                            2           prop = value; // property
                            3           constructor(...) { // constructor
                            4             // ...
                            5           }
                            6           method(...) {} // method
                            7           get something(...) {} // getter
                            8           set something(...) {} // setter
                            9           [Symbol.iterator]() {} //method with a calculated 
                            10          name (here a symbol)
                            11          // ...
                            12       }

MyClassis technically a function (the one we define as constructor), while methods, getters, and setters are written in MyClass.prototype.
